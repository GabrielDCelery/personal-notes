# Site-Packages Explained

## Question
What is site-packages?

## What is `site-packages`?

**`site-packages`** is the directory where Python stores **third-party packages** you install with `pip` or `uv`.

Think of it as Python's library shelf - when you `pip install something`, it goes here.

## What's Inside `site-packages`?

When you run `uv pip install ollama`, it downloads and extracts:

1. **`ollama/`** - The actual Python package (code you import)
   ```python
   import ollama  # Python finds this in site-packages/ollama/
   ```

2. **`ollama-0.6.1.dist-info/`** - Metadata about the package
   - Version info
   - Dependencies
   - Entry points
   - Installation records

## The Complete Picture

### System vs Virtual Environment

**Global site-packages** (system Python):
```
/usr/lib/python3.14/site-packages/
```
Shared across all projects using this Python version.

**Virtual environment site-packages** (your project):
```
/home/gaze/projects/github-GabrielDCelery/youtube-ai-notetaker/.venv/lib/python3.14/site-packages/
```
Isolated to just this project!

## How Python Finds Packages

When you do `import ollama`, Python:

1. Checks built-in modules
2. Searches directories in `sys.path`:
   ```python
   sys.path = [
       '',  # Current directory
       '/usr/lib/python3.14',  # Standard library
       '/path/to/site-packages',  # Third-party packages
   ]
   ```
3. Finds `ollama/` in site-packages
4. Loads `ollama/__init__.py`

## The Hierarchy

```
Python Installation
│
├── lib/python3.14/              # Standard library
│   ├── os.py                    # Built-in: import os
│   ├── json.py                  # Built-in: import json
│   ├── collections/             # Built-in: import collections
│   │
│   └── site-packages/           # Third-party packages
│       ├── ollama/              # pip/uv installed: import ollama
│       ├── rich/                # pip/uv installed: import rich
│       ├── cv2/                 # pip/uv installed: import cv2
│       └── your_package/        # After: uv pip install -e .
```

## Different Types of site-packages

### 1. User site-packages
```bash
~/.local/lib/python3.14/site-packages/
```
When you `pip install --user package` (without venv)

### 2. Virtual environment site-packages
```bash
your-project/.venv/lib/python3.14/site-packages/
```
When you use virtual environments (isolated per project)

### 3. System site-packages
```bash
/usr/lib/python3.14/site-packages/
```
System-wide installations (usually managed by OS package manager)

## Why Virtual Environments Matter

**Without virtual environments:**
```bash
pip install ollama==0.6.0  # Project A needs this version
pip install ollama==0.7.0  # Project B needs this - OVERWRITES Project A!
```

**With virtual environments:**
```bash
# Project A
cd project-a
python -m venv .venv
source .venv/bin/activate
pip install ollama==0.6.0  # Goes to project-a/.venv/lib/.../site-packages/

# Project B
cd project-b
python -m venv .venv
source .venv/bin/activate
pip install ollama==0.7.0  # Goes to project-b/.venv/lib/.../site-packages/
```

Each project has its own isolated `site-packages`!

## Editable Installs and site-packages

When you do `uv pip install -e .`:

**Normal install** copies files:
```
site-packages/
└── youtube_notetaker/
    ├── __init__.py
    ├── cli.py
    └── downloader.py  # COPIED from your source
```

**Editable install** creates a link:
```
site-packages/
├── youtube-ai-notetaker.egg-link  # Contains path to your source
└── __editable__.youtube_notetaker...  # Modern format
```

Python follows the link to your actual source code directory.

## Summary

- **`site-packages`** = where `pip`/`uv` install third-party packages
- **Standard library** = built into Python (no installation needed)
- **Your code** = in project directory until you `install -e .` it
- **Virtual environments** = give each project its own `site-packages`

It's like:
- **Standard library** = Books that come with your bookshelf
- **site-packages** = Books you bought and added
- **Your project** = The book you're writing
- **Editable install** = Putting your draft book on the shelf while you edit it
