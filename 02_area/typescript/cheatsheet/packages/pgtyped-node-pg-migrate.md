# pgtyped + node-pg-migrate

```sh
npm install pg @pgtyped/runtime
npm install -D @pgtyped/cli node-pg-migrate @types/pg
```

> Raw SQL stack for TypeScript. `node-pg-migrate` runs schema migrations from SQL files. `pgtyped` reads your `.sql` query files and generates TypeScript types — so your queries are type-safe without an ORM.

## Project Structure

```
src/
  db/
    migrations/          # node-pg-migrate migration files
      001_create_users.sql
      002_create_posts.sql
    queries/             # pgtyped query files
      users.sql
      posts.sql
    queries.types/       # auto-generated by pgtyped (do not edit)
      users.types.ts
      posts.types.ts
    client.ts            # pg pool setup
```

## Database Client

```typescript
// src/db/client.ts
import { Pool } from "pg";

export const pool = new Pool({
  connectionString: process.env.DATABASE_URL,
});
```

## Migrations (node-pg-migrate)

### Config

```typescript
// package.json
{
  "scripts": {
    "migrate": "node-pg-migrate up",
    "migrate:down": "node-pg-migrate down",
    "migrate:create": "node-pg-migrate create"
  },
  "node-pg-migrate": {
    "migrationsTable": "pgmigrations",
    "dir": "src/db/migrations",
    "databaseUrl": "${DATABASE_URL}"
  }
}
```

Or via `database.json`:

```json
{
  "defaultEnv": "default",
  "default": {
    "url": { "ENV": "DATABASE_URL" }
  }
}
```

### Writing Migrations

```sh
# Create a new migration file
npx node-pg-migrate create create_users
# → src/db/migrations/1711234567_create_users.sql
```

```sql
-- src/db/migrations/1711234567_create_users.sql

-- Up Migration
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  email TEXT NOT NULL UNIQUE,
  role TEXT NOT NULL DEFAULT 'user',
  active BOOLEAN NOT NULL DEFAULT true,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- Down Migration
DROP TABLE users;
```

```sql
-- src/db/migrations/1711234568_create_posts.sql

CREATE TABLE posts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  title TEXT NOT NULL,
  content TEXT,
  author_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- Down Migration
DROP TABLE posts;
```

### Running Migrations

```sh
npx node-pg-migrate up          # apply all pending
npx node-pg-migrate down        # rollback last
npx node-pg-migrate down 3      # rollback last 3
npx node-pg-migrate redo        # rollback + re-apply last
```

### Programmatic Migration (e.g. in CI or app startup)

```typescript
import runner from "node-pg-migrate";

await runner({
  databaseUrl: process.env.DATABASE_URL!,
  dir: "src/db/migrations",
  direction: "up",
  migrationsTable: "pgmigrations",
});
```

## Query Types (pgtyped)

### Config

```yaml
# pgtyped.config.yaml
transforms:
  - mode: sql
    include: "src/db/queries/**/*.sql"
    emitTemplate: "{{dir}}/{{name}}.types.ts"

srcDir: src/
failOnError: true
camelCaseColumnNames: true

db:
  host: localhost
  port: 5432
  dbName: mydb
  user: postgres
  password: postgres
```

### Writing Queries

```sql
-- src/db/queries/users.sql

/* @name GetUserById */
SELECT * FROM users WHERE id = :id!;

/* @name GetActiveUsers */
SELECT id, name, email FROM users
WHERE active = true
ORDER BY created_at DESC
LIMIT :limit!;

/* @name InsertUser */
INSERT INTO users (name, email, role)
VALUES (:name!, :email!, :role!)
RETURNING *;

/* @name UpdateUser */
UPDATE users
SET name = :name, active = :active
WHERE id = :id!
RETURNING *;

/* @name DeleteUser */
DELETE FROM users WHERE id = :id!;
```

```sql
-- src/db/queries/posts.sql

/* @name GetPostsWithAuthor */
SELECT
  p.id,
  p.title,
  p.content,
  p.created_at,
  u.name AS author_name
FROM posts p
INNER JOIN users u ON u.id = p.author_id
WHERE p.author_id = :authorId!;

/* @name InsertPost */
INSERT INTO posts (title, content, author_id)
VALUES (:title!, :content, :authorId!)
RETURNING *;
```

### Generating Types

```sh
# One-time generation (against live DB)
npx pgtyped -c pgtyped.config.yaml

# Watch mode during development
npx pgtyped -c pgtyped.config.yaml -w
```

This generates `*.types.ts` files next to each `.sql` file.

### Using Generated Queries

```typescript
import { pool } from "../client";
import {
  getUserById,
  getActiveUsers,
  insertUser,
  updateUser,
  deleteUser,
} from "./queries/users.types";

// Select one
const user = await getUserById.run({ id: "abc-123" }, pool);
// user is typed as IGetUserByIdResult[]

// Select many
const users = await getActiveUsers.run({ limit: 10 }, pool);

// Insert
const [newUser] = await insertUser.run(
  { name: "Alice", email: "alice@example.com", role: "user" },
  pool,
);

// Update
const [updated] = await updateUser.run(
  { id: "abc-123", name: "Alice Smith", active: true },
  pool,
);

// Delete
await deleteUser.run({ id: "abc-123" }, pool);
```

### Transactions

```typescript
import { pool } from "../client";
import { insertUser } from "./queries/users.types";
import { insertPost } from "./queries/posts.types";

const client = await pool.connect();
try {
  await client.query("BEGIN");

  const [user] = await insertUser.run(
    { name: "Alice", email: "alice@example.com", role: "user" },
    client,
  );

  await insertPost.run(
    { title: "First Post", content: null, authorId: user.id },
    client,
  );

  await client.query("COMMIT");
} catch (err) {
  await client.query("ROLLBACK");
  throw err;
} finally {
  client.release();
}
```

## Workflow Summary

```sh
# 1. Create and apply a migration
npx node-pg-migrate create add_users_table
# edit the generated .sql file
npx node-pg-migrate up

# 2. Write queries in .sql files, then generate types
npx pgtyped -c pgtyped.config.yaml

# 3. Import and use the typed query functions in your code
```
